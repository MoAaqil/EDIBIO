------------------------------------------------------------------------
- Root directory cleaned: Deleted unused log and compile output files.

- Integrated "copy-backend.js": Since Next.js requires TypeScript imports to be local, this pre-build script copies Mongoose models from the backend directory to the frontend library directory prior to compilation.
- Refactored package.json scripts to automate copying.

6. RECENT UPDATE LOG (JULY 2026)
------------------------------------------------------------------------
- Fixed account switching synchronization bug on mobile devices by ensuring that `isHydrating` is reset to false upon store reset (logout), permitting subsequent user logins to fetch cloud data immediately.
- Prevented race conditions that overwrite cloud database state with empty/stale local state by gating push synchronization until the first cloud pull completes successfully.
- Resolved build failures on Vercel by modifying copy-backend scripts to preserve existing committed libraries and using fallback Firebase credentials during static page prerendering.
- Dynamic Storefront URL: Replaced hardcoded localhost:3001 API endpoints in Edibio Online Store page with a dynamic utility resolving to https://edistore.vercel.app in production, and falling back to localhost when running locally.
- Real-Time MongoDB Database Connection: Aligned models so that Edibio billing app writes listings/read orders directly in the 'edistore' database, sharing the same live collections as the marketplace storefront app.
